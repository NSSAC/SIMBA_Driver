SIMBA Scheduler

Type: array of object

The schema describes which and how modules are executed during each tick of the driver.

Each item of this array must be:

root Module Schedule
Type: object

A specification of a module and its schedule.

root Module Schedule name
Type: string

The name of the module.

root Module Schedule class
Type: string

The python class name of the module.

root Module Schedule package
Type: string

The python package name of the module.

root Module Schedule priority

Higher priority is executed first. Priorities must be unique.

  • Option 1
  • Option 2
root Module Schedule priority anyOf item 0
Type: number
root Module Schedule priority anyOf item 1
Type: enum (of string)

Must be one of:

  • "-Infinity"
  • "Infinity"

root Module Schedule startTick
Type: integer Default: 0

The tick the module is executed the first time.

root Module Schedule tickIncrement
Type: number Default: 0

The increment determining the next tick at which the module is executed. If the increment is 0 the module is only executed once.

Value must be greater or equal to 0

root Module Schedule endTick
Default: "startTick if tickIncrement is 0, Infinity otherwise."

The last Tick at which the Module may be executed.

  • Option 1
  • Option 2
root Module Schedule endTick anyOf item 0
Type: integer
root Module Schedule endTick anyOf item 1
Type: enum (of string)

Must be one of:

  • "Infinity"

root Module Schedule moduleData
Type: object

Module specific data provided to the module upon execution.
